# kiberone-reactivation-landing

Мини-лендинг под кампанию реактивации базы «пробник без договора» (2143 контакта).
Один файл `index.html` без зависимостей и без бэкенда — все заявки уходят
в WhatsApp с предзаполненным текстом через `wa.me/...`.

## Структура

- **Hero** — заголовок «новый формат», бейдж «Новый формат · Сентябрь 2026», блок персонализации (по UTM)
- **Countdown-полоса** — авто-таймер «до 1 сентября осталось N дней»
- **Видео-фасад под 60-сек ролик** — YouTube-facade (lazy), клик подгружает iframe. Пока `data-video-id` пустой — кнопка ведёт к CTA-форме.
- **«Что именно изменилось»** — 4 карточки «Было → Стало» в цифрах (время, инструменты, навык, результат)
- **Before/After** — сравнение старого и нового пробного
- **3 CTA** в порядке убывания готовности:
  1. 🟡 Записаться на AI-пробный (модалка с 3 полями → передаёт данные в Камиллу)
  2. 🔵 Спросить у Камиллы (открывает встроенный чат-виджет с preset-сообщением — про расписание в hero, про общий вопрос в финале)
  3. 🖤 Позвонить (`tel:+79274334966` — единый многоканальный номер KIBERone, распределяется по хантерам)
- **Блок «Что нового»** — 3 буллета (сценарий с ИИ / озвучка / готовая игра за 1 час)
- **Проекты детей** — карусель со скринами
- **Отзывы** — 3 карточки родителей
- **Гарантия** — «если не зайдёт — мы больше не беспокоим»
- **FAQ** — 7 частых вопросов реактивационных родителей (details/summary аккордеон)
- **AI-ассистент Камилла** — sticky-виджет справа снизу, `https://assistant.it-kiber.ru`. Держит родителя на странице и передаёт лид в CRM без ручной обработки.
- **Sticky mobile bar** — жёлтая кнопка «Записаться» + WA-кнопка внизу экрана на мобиле, появляется после 400px скролла
- **Footer** — список городов

## Аналитика

Подключены Яндекс.Метрика `109689627` (тот же счётчик, что у kanikuly) и Top.Mail.Ru для VK Ads `3770897`. Все клики шлют цели в оба, через хелпер `window.reachGoal(name, params)`.

**JS-цели** — в интерфейсе Метрики (Настройка → Цели → Тип: JavaScript-событие) заведите:

| Идентификатор | Событие |
|---|---|
| `open_modal` | клик по любой кнопке «Записаться» |
| `submit_lead` | отправка формы (главная цель) |
| `click_ask_schedule` | клик «Спросить у Камиллы» в hero |
| `click_ask_manager` | клик «Спросить у Камиллы» в финальном CTA |
| `click_call` | клик «Позвонить» / клик по номеру в мобильной панели |
| `click_header_cta` | клик по кнопке в шапке |
| `open_kamilla` | родитель открыл чат с Камиллой |
| `video_play` | клик по кнопке play в видео-фасаде |
| `scroll_50` / `scroll_75` / `scroll_100` | глубина просмотра |

В VK Ads (Top.Mail.Ru) те же идентификаторы будут прилетать автоматически — настройте их как события в кабинете VK Ads → «События сайта».

## UTM-параметры для персонализации

При запуске рекламы в VK Ads / Яндекс / SMS / WhatsApp ссылка должна быть вида:

```
https://landing.it-kiber.ru/?name=Айнур&child=Тимур&last=ноябрь%202024&city=Челны
```

| Параметр | Что делает |
|---|---|
| `name` | подставляется в обращение и в форму записи |
| `child` | имя ребёнка — попадает в hero и в WhatsApp-сообщение |
| `last` | период прошлого пробного (формирует текст «в [месяц год] [ребёнок] был у нас») |
| `city` | определяет, в какой филиальный WhatsApp ведёт кнопка |

⚠️ Городские параметры (lowercase): `челны`, `нижнекамск`, `казань`, `елабуга`, `краснодар`, `сургут`, `пермь`.

Если параметров нет — лендинг работает в общем режиме без персонализации, кнопки ведут на центральный WhatsApp (`DEFAULT_PHONE` в коде).

## Что должен сделать маркетолог перед стартом волны

1. **Телефон** — единый номер для входящих звонков `+7 (927) 433-49-66` зашит в кнопках «Позвонить» и мобильной панели. WhatsApp-переходов на сайте нет: WA Камиллы `+79370475149` пока в бане Meta, старый Wazzup-канал занят под B2C. Вместо wa.me все запросы идут во встроенный виджет Камиллы (тот же Wazzup+CRM, только не покидая страницу).
2. **Подставить YouTube ID видео** — атрибут `data-video-id` у `<div id="video-facade">` (ищите комментарий `⚠️ Маркетолог`). Опционально положить `poster.jpg` 1200×675 в корень — тогда стилизовать фасад через `background-image` в CSS блока `.video-facade`.
3. **Домен** — уже `reactivation.it-kiber.ru`.
4. **Метрика + VK Ads** — уже подключены (см. блок «Аналитика»). Осталось только завести JS-цели в кабинете Метрики.
5. **Подготовить UTM-таблицу** — для каждого контакта из базы своя ссылка с подставленным `name/child/last/city`. Генерируется скриптом (см. ниже).
6. **`og-image.png` 1200×630** — уже сгенерирован (`scratchpad/gen_og.ps1`, PowerShell + GDI+). Если нужно перерисовать под новую кампанию — правьте текст в скрипте и запускайте `& gen_og.ps1`.

## Деплой

Простейший вариант — Cloudflare Pages (как `kiber-summer-landing`):
1. Создать репо на GitHub `kiberone-reactivation-landing`
2. В Cloudflare Pages — Connect to Git → выбрать репо → root directory `/`
3. Получить домен `reactivation-landing.pages.dev` → CNAME на свой `reactivation.it-kiber.ru`
4. На push в `main` — автодеплой

Альтернатива — статика на любом хостинге, файл один.

## Генератор персональных ссылок

Готовый скрипт — `salary_bot/gen_personal_links.py`. Забирает `trial_dropoffs_<дата>.xlsx`, дописывает колонку «Personal Link» и выдаёт на выхлопе два CSV:

- `*_for_sms.csv` — короткий (phone, link) для SMS-провайдера
- `*_for_whatsapp.csv` — расширенный (phone, parent, child, city, link, message) для Wazzup24

```bash
cd C:\Users\Honor\salary_bot
python gen_personal_links.py                                  # берёт последний trial_dropoffs_*.xlsx
python gen_personal_links.py trial_dropoffs_2026-07-01.xlsx   # конкретный файл
python gen_personal_links.py --base https://my.domain.ru/     # свой домен
```

Формат итоговой ссылки:
```
https://reactivation.it-kiber.ru/?name=Айгуль&child=Тимур&city=Челны&last=ноябрь%202024
```

Источник самой базы — `salary_bot/trial_dropoffs_report.py`.
